# Диспетчер параллельной обработки файлов (ОС и СП)

## 1. Назначение проекта
Программа `dispatcher` реализует диспетчер параллельных процессов обработки файлов во входном каталоге.

Логика работы:
1. Диспетчер читает список входных файлов по расширению.
2. Запускает до `N` параллельных дочерних процессов.
3. Каждый дочерний процесс запускает внешний скрипт обработки через `execve()` и получает имя входного файла.
4. Как только один процесс завершился, диспетчер запускает новый процесс для следующего файла.
5. После завершения обработки всех файлов запускается финальный внешний скрипт через `execve()` для сборки итогового результата.

## 2. Структура проекта

### 2.1 Заголовочные файлы (`include/`)
- `include/options.h` — тип параметров `options_t` и API разбора/проверок опций.
- `include/string_list.h` — тип динамического списка строк `string_list_t` и API работы со списком.
- `include/file_collector.h` — API сбора входных файлов из каталога.
- `include/process_dispatcher.h` — API диспетчеризации процессов и финального запуска.
- `include/memory_utils.h` — API безопасной работы с динамической памятью.

### 2.2 Исходные файлы (`src/`)
- `src/main.c` — точка входа и оркестрация сценария.
- `src/options.c` — разбор аргументов, проверка скриптов и создание каталога.
- `src/string_list.c` — динамический список строк.
- `src/file_collector.c` — чтение каталога, фильтрация по расширению.
- `src/process_dispatcher.c` — параллельный запуск обработки и финальный запуск.
- `src/memory_utils.c` — функции `xmalloc/xrealloc/xstrdup`.

### 2.3 Скрипты
- `scripts/process_file.sh` — обработка одного файла (входной файл передаётся аргументом).
- `scripts/finalize_djvu.sh` — сборка итогового `.djvu` (выходной файл передаётся через `-o/--output`).

### 2.4 Прочее
- `Makefile` — сборка и очистка (`all`, `clean`).
- `.gitignore` — исключения для временных/служебных файлов.

## 3. Зависимости
Для выполнения скриптов нужны утилиты:
- `c44`, `djvm` (пакет **djvulibre**);
- `tifftopnm` (пакет **netpbm**) или `gm` (GraphicsMagick) / `convert` (ImageMagick).

## 4. Сборка
```bash
make all
```

Флаги компиляции:
- `-std=c11`
- `-Wall -Wextra -Werror`
- `-pedantic`
- `-D_POSIX_C_SOURCE=200809L`

## 5. Очистка
```bash
make clean
```

## 6. Запуск
```bash
./dispatcher -i <input_dir> -o <output_file> [опции]
```

Опции:
- `-i <dir>` — каталог входных файлов (обязательно).
- `-o <file>` — итоговый выходной файл (обязательно).
- `-p <path>` — путь к скрипту обработки одного файла.
- `-f <path>` — путь к финальному скрипту.
- `-d <dir>` — каталог промежуточных `.djvu`.
- `-e <ext>` — расширение входных файлов (по умолчанию `.tiff`).
- `-w <num>` — число параллельных процессов.
- `-h` — справка.

Пример:
```bash
./dispatcher -i ./input_scans -o ./result/book.djvu -w 4 -e .tiff
```

## 7. Функциональная структура проекта

### 7.1 Модуль `src/main.c`
1. `int main(int argc, char **argv)`
- Назначение: общий сценарий запуска диспетчера.
- Принимает: аргументы командной строки.
- Возвращает: `EXIT_SUCCESS`/`EXIT_FAILURE`.

### 7.2 Модуль `src/options.c`
1. `void print_usage(const char *program_name)`
- Назначение: вывод справки.
- Принимает: имя программы.
- Возвращает: ничего.

2. `void options_init(options_t *options)`
- Назначение: установка значений по умолчанию.
- Принимает: структуру параметров.
- Возвращает: ничего.

3. `void options_free(options_t *options)`
- Назначение: освобождение полей структуры параметров.
- Принимает: структуру параметров.
- Возвращает: ничего.

4. `int parse_options(int argc, char **argv, options_t *options, int *help_requested)`
- Назначение: разбор опций через `getopt`.
- Принимает: `argc`, `argv`, структуру параметров, флаг справки.
- Возвращает: `0` при успехе, `-1` при ошибке.

5. `int ensure_script_executable(const char *script_path)`
- Назначение: проверка исполняемости скрипта.
- Принимает: путь к скрипту.
- Возвращает: `0`/`-1`.

6. `int ensure_directory(const char *directory_path)`
- Назначение: создание каталога (включая промежуточные части пути).
- Принимает: путь к каталогу.
- Возвращает: `0`/`-1`.

7. `int clean_intermediate_files(const char *directory_path, const char *extension)`
- Назначение: очистка каталога промежуточных файлов перед новым запуском.
- Принимает: путь к каталогу и расширение.
- Возвращает: `0`/`-1`.

8. `static int parse_worker_count(const char *text_value, size_t *worker_count)`
- Назначение: проверка и преобразование значения `-w`.
- Принимает: строку с числом, указатель результата.
- Возвращает: `0`/`-1`.

### 7.3 Модуль `src/string_list.c`
1. `void string_list_init(string_list_t *list)`
- Назначение: инициализация списка строк.
- Принимает: структуру списка.
- Возвращает: ничего.

2. `void string_list_push(string_list_t *list, char *value)`
- Назначение: добавление строки в список.
- Принимает: список и строку.
- Возвращает: ничего.

3. `void string_list_free(string_list_t *list)`
- Назначение: освобождение списка и его элементов.
- Принимает: список.
- Возвращает: ничего.

4. `void string_list_sort(string_list_t *list)`
- Назначение: сортировка списка.
- Принимает: список.
- Возвращает: ничего.

5. `static int compare_strings(const void *left, const void *right)`
- Назначение: компаратор для `qsort`.
- Принимает: два указателя на элементы массива.
- Возвращает: результат сравнения.

### 7.4 Модуль `src/file_collector.c`
1. `int collect_input_files(const char *input_dir, const char *extension, string_list_t *files)`
- Назначение: сбор входных файлов каталога по расширению.
- Принимает: каталог, расширение, список-накопитель.
- Возвращает: `0`/`-1`.

2. `static char *join_path(const char *left, const char *right)`
- Назначение: объединение пути каталога и имени файла.
- Принимает: две части пути.
- Возвращает: новый путь.

3. `static int has_extension(const char *file_name, const char *extension)`
- Назначение: проверка расширения имени файла.
- Принимает: имя файла, расширение.
- Возвращает: `1`/`0`.

### 7.5 Модуль `src/process_dispatcher.c`
1. `int run_dispatcher(const options_t *options, const string_list_t *files)`
- Назначение: распределение файлов между дочерними процессами и контроль завершений.
- Принимает: параметры запуска, список входных файлов.
- Возвращает: `0`/`-1`.

2. `int run_final_script(const options_t *options)`
- Назначение: запуск финального скрипта сборки через `execve`.
- Принимает: параметры запуска.
- Возвращает: `0`/`-1`.

3. `static size_t find_free_slot(const worker_slot_t *slots, size_t slot_count)`
- Назначение: поиск свободного слота рабочего процесса.
- Принимает: массив слотов и его размер.
- Возвращает: индекс свободного слота или `slot_count`.

4. `static size_t find_slot_by_pid(const worker_slot_t *slots, size_t slot_count, pid_t pid)`
- Назначение: поиск слота по `pid`.
- Принимает: массив слотов, размер, `pid`.
- Возвращает: индекс или `slot_count`.

5. `static int wait_for_child(pid_t *finished_pid, int *status_value)`
- Назначение: ожидание завершения дочернего процесса.
- Принимает: указатели для `pid` и статуса.
- Возвращает: `0`/`-1`.

6. `static int spawn_processing_process(const options_t *options, const char *input_file, pid_t *child_pid)`
- Назначение: запуск процесса обработки файла через `fork + execve`.
- Принимает: параметры, путь входного файла, указатель для `pid`.
- Возвращает: `0`/`-1`.

7. `static char **build_child_env(const options_t *options)`
- Назначение: формирование окружения для дочерних процессов.
- Принимает: параметры запуска.
- Возвращает: массив окружения для `execve`.

8. `static char *make_env_assignment(const char *name, const char *value)`
- Назначение: создание строки переменной окружения `NAME=VALUE`.
- Принимает: имя и значение.
- Возвращает: динамическую строку.

9. `static void free_child_env(char **child_env)`
- Назначение: освобождение массива окружения.
- Принимает: массив окружения.
- Возвращает: ничего.

### 7.6 Модуль `src/memory_utils.c`
1. `void *xmalloc(size_t size)`
- Назначение: безопасный `malloc`.
- Принимает: размер блока.
- Возвращает: указатель на память.

2. `void *xrealloc(void *memory, size_t size)`
- Назначение: безопасный `realloc`.
- Принимает: исходный указатель и новый размер.
- Возвращает: новый указатель.

3. `char *xstrdup(const char *source)`
- Назначение: безопасное дублирование строки.
- Принимает: исходную строку.
- Возвращает: новый буфер строки.

### 7.7 Внешние скрипты
1. `scripts/process_file.sh`
- Назначение: обработка одного входного файла и создание промежуточного `.djvu`.
- Принимает: позиционный аргумент `input_file`.
- Возвращает: код `0` при успехе, ненулевой код при ошибке.

2. `scripts/finalize_djvu.sh`
- Назначение: сборка итогового `.djvu` из промежуточных файлов.
- Принимает: опцию `-o|--output <output_file>`.
- Возвращает: код `0` при успехе, ненулевой код при ошибке.

## 8. Примечания по соответствию требованиям
- Параметры командной строки разбираются через `getopt`.
- При запуске без параметров печатается справка.
- Внешние скрипты запускаются строго через `execve()`.
- Все системные вызовы и результаты `malloc/realloc` проверяются.
- При ошибке выделения памяти в неинтерактивной программе вызывается `abort()`.
- Используются только идентификаторы в `snake_case`, макросы в верхнем регистре.
