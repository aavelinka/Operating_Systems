# Лабораторная работа №2: Понятие процессов

## Состав проекта

- `parent` — родительский процесс.
- `child` — дочерний процесс (запускается через `execve()`).
- `env` — список имён переменных окружения для сокращённой среды.

Исходники:

- `src/parent.c`
- `src/child.c`
- `src/env_utils.c`
- `include/env_utils.h`
- `Makefile`

## Сборка

```bash
make
```

Ключи компиляции:

- `-W -Wall -Wextra -std=c11 -pedantic`
- Дополнительно: `-Wno-unused-parameter -Wno-unused-variable`

Очистка:

```bash
make clean
```

## Подготовка окружения

Перед запуском `parent` можно задать путь к каталогу с `child`:

```bash
export CHILD_PATH=.
```

Если `CHILD_PATH` не установлен, используется текущий каталог.

Файл `env` должен содержать имена переменных (по одной в строке), например:

- `SHELL`
- `HOME`
- `HOSTNAME`
- `LOGNAME`
- `LANG`
- `TERM`
- `USER`
- `LC_COLLATE`
- `PATH`

## Запуск

```bash
./parent
```

После старта `parent`:

1. Выводит своё окружение, отсортированное при `LC_COLLATE=C`.
2. Читает файл `env`.
3. Формирует сокращённую среду для дочернего процесса.
4. Читает символы из `stdin`:

- `+` — создать дочерний процесс и запустить `child` через `fork()+execve()`, `child` выводит среду через `environ`.
- `*` — аналогично, но `child` выводит среду через третий параметр `main` (`envp`).
- `q` — завершить `parent` после ожидания всех дочерних процессов.

## Аргументы `child`

При запуске из `parent`:

- `argv[0]` имеет вид `child_XX` (`00..99`, номер инкрементируется).
- `argv[1]` — режим: `"+"` или `"*"`.
- Среда передаётся через третий аргумент `execve()` (сформированный массив `envp`).

## Быстрый пример

```bash
printf '+*q' | CHILD_PATH=. ./parent
```

Команда создаст два дочерних процесса (`child_00`, `child_01`), затем завершит `parent`.
